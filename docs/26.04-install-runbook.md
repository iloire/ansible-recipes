# Pepinaco 26.04 install runbook

The day-of-install playbook for moving pepinaco from Ubuntu 22.04 LTS (Jammy) to **Ubuntu 26.04.1 LTS (Resolute Raccoon)** on a fresh disk. Targeted ETA: **September 2026** (`.1` point release, post-trip).

The strategic goal of every commit in this repo since spring 2026 has been: *make this install one-shot*. Things that genuinely can't be ansibled live in the **Manual checklist** at the bottom.

## Pre-install (do these once before install day)

- [ ] **Validate the playbook in a VM.** See [testing.md](testing.md) — the repo ships a `Vagrantfile` that boots Ubuntu 24.04 (faithful 26.04 preview: same libfuse2t64, same modern keyrings, same removal of apt-key) and runs the playbook. `vagrant up && vagrant snapshot save clean`, then iterate. Critical things to verify before install day: `apt_third_party_repos` (every key download succeeds, every repo registers), `libfuse2t64` install (the version-conditional in `roles/apt_minimum`), `code` apt install via the Microsoft repo. Switch to a `bento/ubuntu-26.04` box in the Vagrantfile once it exists (probably summer 2026) for the final pre-install dry-run.
- [ ] **Pin a known-good ansible-recipes commit** so you can `git checkout <sha>` if `master` has drifted. Tag it: `git tag pepinaco-2604-install`.
- [ ] **Verify GitHub SSH access from your Mac** (you'll need to clone repos before the local Linux machine has SSH keys). 1Password Emergency Kit ready.
- [ ] **Backup `~/.ssh/`, `~/.gnupg/`, `~/git-crypt-key`, `~/Documents/`, `~/Pictures/`, `~/Desktop/`, browser profiles** to the DATA drive (`/media/ivan/DATA/backups/pre-2604/`).
- [ ] **Note current state** — run `./scripts/audit-drift.sh > /tmp/pre-install-drift.md` and copy to `Areas/computer systems/machines/pepinaco/` so you can compare post-install.
- [ ] **Confirm disk plan.** Install target is `sdc` (the new Kingston A400, serial `50026B76853D6777`, firmware 66.5). Old Ubuntu on `sdd` stays as fallback. Windows on `sde` untouched. Verify by `lsblk -o NAME,SIZE,SERIAL,TRAN`.

## Install day

### Phase 1: clean OS install

1. **Boot the 26.04.1 USB** — F11 at POST gets the boot menu. Pick the USB device.
2. **In the installer**: choose "Install Ubuntu", manual partitioning, target `sdc`. Standard layout: 1 GB EFI (`/boot/efi`, FAT32), rest as ext4 `/`. **Do not touch sdd/sde.** No swap partition (use a swap file if needed later).
3. **User**: `ivan`, hostname `pepinaco`. Do NOT enable LUKS (already in use on the DATA drive separately if at all).
4. **Reboot.** GRUB should auto-detect Ubuntu on `sdd` and Windows on `sde`. If not, run `sudo update-grub` after first login. F11 at POST also works as escape hatch.

### Phase 2: bootstrap ansible

```bash
# 5. Generate SSH key + add to GitHub from another machine
ssh-keygen -t ed25519 -C "ivan@pepinaco-26.04"
cat ~/.ssh/id_ed25519.pub
# (paste into github.com/settings/keys via your phone or laptop)

# 6. Install ansible
sudo apt update
sudo apt install -y git software-properties-common
sudo apt-add-repository --yes --update ppa:ansible/ansible
sudo apt install -y ansible

# 7. Clone the repo
git clone git@github.com:iloire/ansible-recipes.git ~/code/ansible-recipes
cd ~/code/ansible-recipes
git checkout pepinaco-2604-install   # the pinned tag from pre-install

# 8. Run the full play (target: pepinaco only)
./run.sh --limit pepinaco
```

The first run will take 30–60 minutes. It will pause for sudo password once. If a single role fails, re-run with `--start-at-task='<task name>'` after fixing the issue — most roles are idempotent.

### Phase 3: manual restore (the things ansible can't do)

This is the irreducible "human in the loop" set. Keep it short by design:

- [ ] **Restore SSH keys** beyond the install-day key: `cp -r /media/ivan/DATA/backups/pre-2604/.ssh/* ~/.ssh/ && chmod 600 ~/.ssh/id_*`
- [ ] **Restore GPG keys**: `gpg --import < /media/ivan/DATA/backups/pre-2604/gpg-private.asc` then `gpg --edit-key <id>` and `trust`
- [ ] **Restore git-crypt key**: `cp /media/ivan/DATA/backups/pre-2604/git-crypt-key ~/git-crypt-key`. Then `cd <encrypted repo> && git-crypt unlock ~/git-crypt-key` for each.
- [ ] **1Password sign-in** — Emergency Kit + account password. Then enable browser integration in Brave + Chrome.
- [ ] **Tailscale**: `sudo tailscale up --auth-key=<pre-auth>` (generate a one-time pre-auth key from the Tailscale admin console on another machine first)
- [ ] **NVIDIA proprietary driver**: `sudo ubuntu-drivers autoinstall && sudo reboot`. After reboot: `nvidia-smi` should show the GPU. (Not in ansible because the driver version pin breaks every Ubuntu release; let `ubuntu-drivers` pick what works on 26.04.)
- [ ] **Mount the DATA drive** by UUID. Get UUID: `sudo blkid | grep DATA`. Then add to `/etc/fstab`: `UUID=<uuid> /media/ivan/DATA ext4 defaults 0 0` and `sudo mount -a`. (Not in the playbook because the UUID is host-specific.)
- [ ] **Browser sign-in** — Chrome + Brave + Firefox. Sync handles bookmarks + passwords (1Password handles the rest).
- [ ] **Docker post-install verification**: `newgrp docker && docker run hello-world`. The ansible docker role adds the user to the docker group, but you need a fresh shell session for it to take effect.
- [ ] **GNOME extensions installed via Extension Manager** — open the flatpak Extension Manager (already installed by `flatpak_packages`) and install:
  - `smart-auto-move@khimaros.com` (window position memory — the dconf settings are already in place, just need the extension itself)
  - Any others you decide on. The dconf state for `enabled-extensions` / `disabled-extensions` is already set — installing the extension is enough to activate it.
- [ ] **Conky autostart** — should work via dotfiles xdg/autostart, but if you don't see the system overlay on next login, check `pgrep conky` and the journal: `journalctl --user -u conky -b 0`.
- [ ] **Verify Obsidian.AppImage autostart** — this entry has a hardcoded `/home/ivan/apps/Obsidian.AppImage` path. Either restore that AppImage to that path, or use the snap version (`obsidian` is in `snap_classic_packages`) and edit `~/dotfiles/xdg/autostart/Obsidian.AppImage.desktop` to `Exec=obsidian`.
- [ ] **Verify pactl autostart** — sets default audio sink to `alsa_output.pci-0000_02_00.1.hdmi-stereo-extra2`. Same hardware, so the device name *should* match — if not, get the new name from `pactl list short sinks` and update `~/dotfiles/xdg/autostart/pactl.desktop`.

## Post-install verification

After the manual checklist, run these checks:

```bash
# Quick sanity: every apt third-party repo registered cleanly
ls /etc/apt/keyrings/
ls /etc/apt/sources.list.d/

# Modern CLI tools all in place
which bat btop dust fd delta eza atuin tldr gitleaks lazygit

# VS Code is the apt version, not snap
which code   # should be /usr/bin/code, not /snap/bin/code

# Fonts loaded
fc-list | wc -l   # higher than fresh install means dotfiles fonts loaded
fc-list | grep -i belbo

# Templates symlink active
ls -la ~/Templates   # should be a symlink to ~/dotfiles/Templates

# dconf settings applied
dconf read /org/gnome/desktop/input-sources/xkb-options
# expect: ['caps:escape', 'grp:alt_shift_toggle']

dconf read /org/gnome/nautilus/list-view/use-tree-view
# expect: true

# Run audit-drift again — anything new is a regression to investigate
./scripts/audit-drift.sh > /tmp/post-install-drift.md
diff /tmp/pre-install-drift.md /tmp/post-install-drift.md | less
```

## Rollback plan

If anything is unrecoverable: F11 at POST, pick the old Ubuntu on `sdd`. The install on `sdc` doesn't touch the old root, and GRUB will already have an entry for it. Worst case: boot the 22.04 USB and `chroot` to fix grub if something corrupted the EFI.
